# Subkari（サブカリ）

##  概要
『Subkari』は、洋服に特化した「購入」と「レンタル」の両方が選択できるCtoC（個人間取引）型のフリマアプリケーションです。
従来のフリマアプリにおける「実物を見られないため、サイズ感がわからない」という最大の課題を解決し、ユーザーが安心して好みの subculture ファッションを楽しめるプラットフォームを目指して開発しました。

##  最大のこだわり（課題解決へのアプローチ）
- **「サイズがわからない問題」を解消する詳細入力システム**
  服の種類（トップス、アウター、ボトムス、ワンピースなど）を選択すると、それぞれの種類に応じた細かなサイズ数値（着丈、身幅、肩幅、ウエスト、総丈など）を入力・閲覧できる仕組みを実装しました。これにより、試着ができないCtoC取引でも、手持ちの服と比較して正確なサイズ感を把握することができます。

##  使用技術（技術スタック）
- **フロントエンド**: HTML5, CSS3, JavaScript
- **バックエンド**: Python (Flask / Blueprint機能によるモジュール化)
- **データベース**: MySQL (MAMP環境での構築・運用)
- **バージョン管理**: GitHub

##  主な機能
一般的なフリマ・レンタルサービスに必要な機能を網羅して実装しています。

### 1. ユーザー機能
- **会員登録・ログイン機能**
- **マイページ機能**: プロフィール編集、出品履歴、購入・レンタル履歴の確認。

### 2. 出品・商品管理機能
- **「購入 / レンタル」の選択出品**: 出品時に販売価格だけでなく、レンタル可否や「1泊2日のレンタル料金」を個別に設定可能。
- **服種別の詳細サイズ入力**: カテゴリに連動して、必要なサイズ項目（センチメートル単位）が動的に変化する入力フォーム。
- **画像アップロード・商品検索**: 商品画像付きでの出品、およびキーワードやカテゴリ、サイズ、取引形態（購入/レンタル）による絞り込み検索。

### 3. 取引・決済フロー
- **購入/レンタル手続き**: 在庫ステータスと連動したスムーズな注文処理。
- **取引チャット機能**: 出品者と購入者（レンタル者）の間で、発送や返却に関する連絡を行うためのリアルタイムなコミュニケーションスペース。
- **ステータス管理**: 「出品中」「取引中」「レンタル中」「返却完了」などの状態をデータベース上で厳密に管理。

##  アーキテクチャ / 実装の工夫
- **Flask Blueprintによるモジュール化**:
  アプリケーションの規模拡大（スケーラビリティ）を見据え、ユーザー管理(`auth`)、商品管理(`products`)、取引管理(`transactions`)など、機能ごとにディレクトリとルーティングを論理的に分割。コードの可読性と保守性を高めました。
- **MySQLによる厳密なデータリレーション**:
  「1つの商品が、ある時は購入され、ある時は複数回レンタルされる」という複雑なCtoCの取引状態を破綻なく管理するため、商品テーブル、注文テーブル、レンタルスケジュールテーブルのリレーション（外部キー制約）を適切に設計しました。

##  環境構築・データベースについて
本リポジトリには、環境構築および動作確認のための以下のファイルが含まれています。

- **`environment.yml`**: 仮想環境（Conda等）の構築に必要なパッケージ一式が定義されています。
- **`Subkari_DB.sql`**: アプリケーションの動作に必要なデータベースのテーブル構造（スキーマ）を定義したSQLファイルです。
- **`Subkari_DB_testdata.sql`**: アプリケーションの動作確認やデバッグをスムーズに行うための、初期テストデータが含まれたSQLファイルです。

##  開発体制
- **人数**: 7名によるチーム開発（学校のプロジェクト）
- **自身の役割・担当範囲**: 
  - **プロジェクトマネジメント（全体指揮・チームリーダー）**:
    - チーム全体の進捗管理およびタスクの割り当て。
    - 開発遅延を防ぐための工数見積もりの見直し、および計画の柔軟な変更・修正。
    - 技術的な課題に直面したメンバーのフォロー・補佐（ペアプログラミングやトラブルシューティングなど）。
  - **データベース設計・構築**:
    - MySQL（MAMP環境）を用いた、CtoC取引やレンタルシステムを破綻なく管理するためのリレーション設計およびテーブル構築。
  - **環境整理**:
    - チーム全体がスムーズに開発・動作確認を行えるよう、Conda環境のパッケージを定義した `environment.yml` の作成および初期テストデータ（SQL）の整備。
  - **フロントエンド開発**:
    - 一部画面のUI/UXデザインおよび実装。
